## 一、先确定依赖方向

三个人的代码不应互相直接调用多个内部类，否则联调时容易形成循环依赖。建议固定为：

```text
成员1：Gradio UI
        │
        │ 只调用统一业务接口
        ▼
成员3：RAG / Agent / 文档处理
        │
        │ 只调用图数据库抽象接口
        ▼
成员2：GraphDB 适配器
```

即：

```text
ui → service → graph_repository
```

禁止出现：

```text
ui → GraphDB
GraphDB → RAG
GraphDB → Gradio
RAG → Gradio组件
```

项目要求包含 GraphRAG、Agentic RAG、意图识别和工具调用，因此数据库层应只返回图数据和文档证据，最终答案仍由 RAG/Agent 层生成。

# 二、共享接口层

三个人共同维护一个 `contracts` 目录，专门放接口和数据结构。

```text
project/
├── contracts/
│   ├── models.py
│   ├── backend_service.py
│   ├── graph_repository.py
│   └── errors.py
├── ui/                       # 成员1
├── graphdb/                  # 成员2
├── rag/                      # 成员3
├── tests/
└── app.py
```

`contracts` 中只允许放：

* 数据类
* 枚举
* 接口声明
* 错误码

不能放具体业务实现。

这样成员 1 可以用 Mock 后端开发 Gradio，成员 3 可以用 Mock GraphDB 开发 RAG，成员 2 可以使用固定 JSON 测试 GraphDB，而不用等其他人完成。

# 三、共享数据模型

建议使用标准库 `dataclass`，避免三个成员传递结构不统一的普通字典。

## 1. 文档与知识图谱数据

```python
# contracts/models.py

from dataclasses import dataclass, field
from enum import Enum
from typing import Any


class IntentType(str, Enum):
    NORMAL_CHAT = "normal_chat"
    DOCUMENT_SEARCH = "document_search"
    GRAPH_QUERY = "graph_query"
    UNKNOWN = "unknown"


@dataclass
class DocumentSummary:
    document_id: str
    filename: str
    knowledge_base_id: str
    chunk_count: int
    entity_count: int
    created_at: str


@dataclass
class ChunkRecord:
    chunk_id: str
    document_id: str
    content: str
    page_number: int | None = None
    title: str | None = None
    metadata: dict[str, Any] = field(default_factory=dict)


@dataclass
class EntityRecord:
    entity_id: str
    name: str
    entity_type: str
    aliases: list[str] = field(default_factory=list)


@dataclass
class RelationRecord:
    relation_id: str
    source_entity_id: str
    relation_type: str
    target_entity_id: str
    source_chunk_id: str
    confidence: float = 1.0


@dataclass
class DocumentGraphPayload:
    schema_version: str
    knowledge_base_id: str
    document: DocumentSummary
    chunks: list[ChunkRecord]
    entities: list[EntityRecord]
    relations: list[RelationRecord]
```

`DocumentGraphPayload` 是成员 3 向成员 2 传递数据的核心格式。

成员 3 负责生成：

* 文档块
* 实体
* 关系
* 置信度
* 来源块

成员 2 负责把这些数据转换成 RDF 三元组并写入 GraphDB。

## 2. 查询请求

```python
@dataclass
class QueryRequest:
    query: str
    session_id: str
    knowledge_base_id: str
    selected_document_ids: list[str] = field(default_factory=list)
    top_k: int = 5
    max_hops: int = 2
    enable_query_rewrite: bool = True
    trace_id: str | None = None
```

字段解释：

| 字段                      | 说明          |
| ----------------------- | ----------- |
| `query`                 | 用户原始问题      |
| `session_id`            | 当前对话会话      |
| `knowledge_base_id`     | 当前知识库       |
| `selected_document_ids` | 用户指定的文档范围   |
| `top_k`                 | 最多返回多少个证据块  |
| `max_hops`              | 图检索最多查询几跳   |
| `enable_query_rewrite`  | 是否启用问题改写    |
| `trace_id`              | 本次请求的日志追踪编号 |

## 3. GraphDB 检索结果

```python
@dataclass
class RetrievedChunk:
    chunk_id: str
    document_id: str
    filename: str
    content: str
    page_number: int | None
    score: float


@dataclass
class GraphNode:
    node_id: str
    label: str
    node_type: str


@dataclass
class GraphEdge:
    edge_id: str
    source: str
    target: str
    relation: str
    source_chunk_id: str | None = None


@dataclass
class RetrievalResult:
    rewritten_query: str
    matched_entities: list[EntityRecord]
    chunks: list[RetrievedChunk]
    nodes: list[GraphNode]
    edges: list[GraphEdge]
    paths: list[list[str]]
```

特别注意：

> 成员 2 返回的是 `RetrievalResult`，而不是自然语言答案。

GraphDB 层不应负责：

* 拼 Prompt
* 调用大模型
* 生成最终答案
* 决定回答语气
* 保存 Gradio 聊天历史

```python
# ui/callbacks.py

from contracts.backend_service import BackendService
from contracts.models import QueryRequest


class UICallbacks:

    def __init__(self, backend: BackendService):
        self.backend = backend

    def upload_file(
        self,
        file_path: str,
        knowledge_base_id: str,
    ):
        document = self.backend.ingest_document(
            file_path=file_path,
            knowledge_base_id=knowledge_base_id,
        )

        documents = self.backend.list_documents(
            knowledge_base_id=knowledge_base_id,
        )

        return (
            f"文档 {document.filename} 上传成功",
            documents,
        )

    def send_message(
        self,
        message: str,
        session_id: str,
        knowledge_base_id: str,
        selected_document_ids: list[str],
    ):
        request = QueryRequest(
            query=message,
            session_id=session_id,
            knowledge_base_id=knowledge_base_id,
            selected_document_ids=selected_document_ids,
        )

        response = self.backend.answer(request)

        return (
            response.answer,
            response.sources,
            {
                "nodes": response.graph_nodes,
                "edges": response.graph_edges,
                "paths": response.graph_paths,
            },
        )
```

## GraphDB 层必须保证的规则

### 1. 写入操作幂等

同一个 `document_id` 重复写入时，不能重复创建节点。

建议：

```text
第一次写入：INSERT
第二次写入：更新或覆盖
```

不能变成：

```text
doc_001
doc_001_copy
doc_001_copy_2
```

### 2. 关系必须保留来源

每条关系必须有：

```text
source_chunk_id
document_id
```

否则系统虽然能够展示“实体 A 与实体 B 有关系”，却无法说明这条关系来自哪份文档。

### 3. 查询必须支持文档范围

下面两个查询结果应不同：

```python
document_ids=[]
```

表示查询整个知识库。

```python
document_ids=["doc_001", "doc_003"]
```

表示只能在指定文档中检索。

### 4. 不暴露 SPARQL 给成员 3

错误设计：

```python
graph_repository.execute_sparql("""
    SELECT ?s ?p ?o
    WHERE ...
""")
```

这样成员 3 需要理解 GraphDB 内部结构。

正确设计：

```python
graph_repository.retrieve_context(
    query=query,
    entity_names=entities,
    knowledge_base_id=knowledge_base_id,
    document_ids=document_ids,
)
```

SPARQL 语句完全封装在成员 2 的模块内部。

## 1. 文档上传接口实现

```python
def ingest_document(
    self,
    file_path: str,
    knowledge_base_id: str,
) -> DocumentSummary:

    # 1. 文档解析和分块
    document, chunks = self.document_processor.process(
        file_path=file_path,
        knowledge_base_id=knowledge_base_id,
    )

    # 2. 实体关系抽取
    entities, relations = self.entity_extractor.extract(
        chunks=chunks,
    )

    # 3. 组装共享数据结构
    payload = DocumentGraphPayload(
        schema_version="1.0",
        knowledge_base_id=knowledge_base_id,
        document=document,
        chunks=chunks,
        entities=entities,
        relations=relations,
    )

    # 4. 调用成员2的接口
    result = self.graph_repository.upsert_document_graph(payload)

    return result
```

这个接口中：

* 成员 3 决定抽取哪些实体和关系；
* 成员 2 决定实体和关系如何转成 RDF；
* 成员 1 只获取上传结果。

## 2. 问答接口实现

```python
def answer(
    self,
    request: QueryRequest,
) -> QueryResponse:

    intent = self.intent_router.detect(request.query)

    if intent == IntentType.NORMAL_CHAT:
        answer = self.answer_generator.normal_chat(
            query=request.query,
            session_id=request.session_id,
        )

        return QueryResponse(
            answer=answer,
            intent=intent,
            original_query=request.query,
            rewritten_query=request.query,
            sources=[],
            graph_nodes=[],
            graph_edges=[],
            graph_paths=[],
            session_id=request.session_id,
            trace_id=request.trace_id or "",
        )

    rewritten_query = self.query_rewriter.rewrite(
        request.query
    )

    entity_names = self.entity_extractor.extract_from_query(
        rewritten_query
    )

    retrieval_result = self.graph_repository.retrieve_context(
        query=rewritten_query,
        entity_names=entity_names,
        knowledge_base_id=request.knowledge_base_id,
        document_ids=request.selected_document_ids,
        top_k=request.top_k,
        max_hops=request.max_hops,
    )

    reranked_chunks = self.reranker.rerank(
        query=rewritten_query,
        chunks=retrieval_result.chunks,
    )

    answer = self.answer_generator.generate(
        query=request.query,
        rewritten_query=rewritten_query,
        chunks=reranked_chunks,
        graph_nodes=retrieval_result.nodes,
        graph_edges=retrieval_result.edges,
    )

    return QueryResponse(
        answer=answer,
        intent=intent,
        original_query=request.query,
        rewritten_query=rewritten_query,
        sources=self._build_sources(reranked_chunks),
        graph_nodes=retrieval_result.nodes,
        graph_edges=retrieval_result.edges,
        graph_paths=retrieval_result.paths,
        session_id=request.session_id,
        trace_id=request.trace_id or "",
    )
```

# 八、错误接口

不要直接把底层异常显示给用户，例如：

```text
requests.exceptions.ConnectionError
SPARQLWrapperException
KeyError: parent_id
```

统一定义：

```python
# contracts/errors.py

class ServiceError(Exception):

    def __init__(
        self,
        code: str,
        message: str,
        retryable: bool = False,
        details: dict | None = None,
    ):
        super().__init__(message)
        self.code = code
        self.message = message
        self.retryable = retryable
        self.details = details or {}
```

建议错误码：

| 错误码                        | 含义           |
| -------------------------- | ------------ |
| `INVALID_FILE_TYPE`        | 文件格式不支持      |
| `DOCUMENT_PARSE_FAILED`    | 文档解析失败       |
| `ENTITY_EXTRACTION_FAILED` | 实体关系抽取失败     |
| `GRAPHDB_UNAVAILABLE`      | GraphDB 无法连接 |
| `GRAPH_WRITE_FAILED`       | 图数据写入失败      |
| `GRAPH_QUERY_FAILED`       | 图查询失败        |
| `MODEL_CALL_FAILED`        | 模型调用失败       |
| `EMPTY_RETRIEVAL_RESULT`   | 未检索到证据       |
| `INVALID_REQUEST`          | 请求参数错误       |

Gradio 层统一处理：

```python
try:
    response = backend.answer(request)
except ServiceError as error:
    return f"操作失败：{error.message}"
```

# 十、接口冻结规则

建议第一天就冻结以下内容：

```text
DocumentGraphPayload
QueryRequest
RetrievalResult
QueryResponse
BackendService
GraphRepository
错误码
```

后续成员不能直接修改对方已经使用的字段。

需要增加字段时：

```python
# 正确：增加带默认值的可选字段
language: str = "zh-CN"
```

不要这样修改：


```text
成员1只认识 BackendService
```
